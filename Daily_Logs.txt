Day 0:
- compile process: preprocessor -> compiler -> produces an executable for you to run.

Day 1: ch. 1 - 10
- C orginally didn't have booleans
- Pre vs Post Increment/Decrement: i++ increments after expression is evaluated; ++i increments before.
- sizeof: size in bytes of the type of expression
- do-while: Test happens at the end of the block instead of the beginning.
- EVERYTHING passed into a function is copied  into its parameter and the function operates on that local copy.
- Always add void to function parameters. Ex: void function_name(void). void funtion_name() is different, and turns off all the comlpiler type checking!
- Pass a pointer to a function -> pointer gets copied! Function can now dereference the original variable.
- For multidimentional arrays, compiler must know all dimensions apart from the 1st dimension.
- You can initialize strings via a pointer or as an array! char *s = "Hello"; char s[] = "Hello";
- For struct pointers, you must de-pointer it first to get to the struct itself: (*c).price = new_price; c->price = new_price;
- Different architectures represent data in memory in different ways! When you write binary data to a stream, serialize it to keep it portable for different achitectures!
- typedef to make an alias for a type: typedef int antelope; // Make "antelope" an alias for "int"

Day 2: ch. 11 - 16
- pointers in an array know exactly how many bytes to skip to arrive at the next index: *(p + 1) // prints the value in index 1. 
- In an array, you can subtract the beginning pointer and end pointer to get the length
- Array/Pointer Equivalence: a[b] = *(a + b) //because a is simply a pointer to index 0
- Cannot copy an array normally - must use a loop or memcpy()
- void* is basically a Generic Type. Ex void *memcpy(void *s1, void *s2, size_t n); //pointer, pointer to destination, number of bytes to copy, works for all data types.
- void pointers can't dereference. Simply set the void pointer to an actual type: void *p = &a; char *q = p;
- Clarification: *p refers to the value p points to. p refers to the address stored inside p.
- Allocation functions return a pointer to the newly-allocated memory: *p = malloc(sizeof(int));
- These also return NULL if memory cannot be allocated, in which you need error-handling: if ((x = malloc(sizeof(int))*100000)==NULL) {//ERROR HANDLING}
- free(p) frees the memory that p points to. If many pointers point to it, freeing one is enogugh.
- char is an int (0-255) and can be used interchangeably
- const int *p; // Can't modify the actual thing. int *const p; // Can't modify the pointer but can modify the actual thing it points to
- If you define a variable const, also define all pointers to it const too
- volatile: imforms a value might change outside of the program: must be checked regularly and should not be optimized. Ex: sensor updates on hardware
- static in block scope: initialized one time and persists between calls.
- extern: use variables from a different file

Day 3: ch.17 - 22
- Never ignore warnings in C; address them all
- To use functions from a different file, create a headerfile (same_name.h) and stick the prototype in it (Ex: int add(int, int); and #include "same_name.h"
- Make sure to only #include a headerfile once (Ex. headerfile #includes more headerfiles): There's a solved way to do this.
- Object file is an in-between file that's not yet an executable. Compiling these are slow but converting to exe is fast, so if you have many files, you can compile them to object first, then to exe. If you modify one of those files, you only have to rebuild that ONE object file.
- When you run a program, you ask the shell, which asks the OS to make a new process and run it
- When modifying a string, usually, you shouldn't make it longer than it already is
- When you run a program, you are running it from another program! (Usually a shell that just runs programs).
- You can use different error exit statuses and script the shell to make decisions on what to do next!
- You can access env variable from inside a c program: getenv()
- Macros: define it and it basically replaces every other place with its name with its corresponding value during preprocessing, before the compiler even sees it
- If defined (#ifdef, #endif): decides whether to include certain chunk of code depending on if a macros was defined
- There are built in macros you can use like date, time, file, etc. Macros can also be arguments
- Padding: some # of bytes your system adds between data, such as fields in a struct. You never know how much padding there is!
- Order matters a lot in C because it's pretty much the same order that's stored in memory: Ex: Pointer to a struct is the same as pointer to the 1ST element in the struct. Ex: You can compress adjacent bytes together (int a:5; int b:5;) to save memory
- enums: Basically a struct but for const int values

Day 4: ch.23 - 33
- You can declare a pointer to the 1st instruction of a function, but you must specify the parameter and return types: float (*p)(int, int);
- You can all a function, or refer to a function via its pointer
- Bit operations: NOT, AND, OR, XOR
- Functions can have variable amount of arguments
- assert() is used to insist that somehting be true of else the program will exit (catch shooudl-never-happen errors)
- Goto lets you skip code until the next label. Commonly use to cleanly break out of a loop (break only breaks out of the innermost loop), or to retry smt.
- Compound literals: create an nameless object on the fly: Ex. int s = sum((int []){1, 2, 3, 4}, 4);
- Generics in C: You specify the possible type and the complier looks at the argument and matches it with one of the types listed. Happens at compile, NOT runtime

WEEK 2: STM32

Day 1:
- APBs (advanced peripheral bus) and AHB (advanced high-performing bus) provide clock access to SP2, I2C, etc. Buses are like roads the data travels on, and some are faster than others
- Peripherals have addresses and inside them, registers have "sub-addresses"
- Addresses of peripherals: In your code, set it up as an offset from the starting address of the bus. Same concept for registers (GPIOA_BASE + SOME_R_OFFSET)
- On registers, you usually have to set certain bit/bits to 1 or 0 depending on the function you want (#define PIN5 (1U<<5))
- Registers have this format: #define RCC_AHB1EN_R (*(volatile unsigned int *)(RCC_BASE + AHB1EN_R_OFFSET)). RCC_AHB1EN_R is now a dereferenced pointer to that address in memory, so now you can edit it its contents
- RCC manages which peripherals get clocks, and also resets that reset peripherals to orginal states. Though RCC in this chip lives inside AHB1, it controls the entire chip.
- 1U<<0 means "The number 1, shifted 0 bits - so ...0001. (1U<<5) means "The number 1, shifted 5 bits - so ...00010 0000 (remember 0 in front of the first bit don't matter but those behind it do matter!)
- Set 2 bits for the MODE_R to define the mode (input, output, alternate, analog)
- When finding the offset address, usually refer to _____ register map.
- All this manual stuff usually gets taken care of in a headerfile

Day 2:
- All GPIO modules have at least 2 registers: Direction register (set pin as input/output) and data register where data is stored (write to or read from pin)
- BSRR: BSy sets the corresponding ODRx bit, BRy resets the corresponding ODRx bit
- if (GPIOC->IDR & BTN_PIN) { The & is a bitwise operator and is different from &&. & compares each bit individually and sees if the resulting value is non-zero

Day 3:
- UART is part of USART where there is both sychronous and asychronous mode and UART is asychronous
- duplex: data can be both transmitted and recieved. simplex means it's just one of them (one direction)
- Unrelated (windows + d) to hide all tabs on OS
- Sometimes (uart for example), you want to reset all the control bits at the start: USART2->CR1 = CR1_TE; use = instead of |=
- The RM contains registers, controls, and stuff so look here when you're enabling stuff and changing the 0s and 1s in bits
- The datasheet contains the structure and wiring of the chip so you usually look here to figure out where pins are
- For UART: 
- UART hardware doesn't take "115200" directly — it needs a specific divisor value written into the BRR: BRR value ≈ PeriphCLK / BaudRate
- Process: set RCC, set GPIOx->MODER, set alternate function type (in this case), configure uart module with RCC, baudrate, transfer direction, and enable uart module

Systick:
- Systick is a core peripheral that's on all ARM microcontrollers, regarless of brand
- It's a downcounter; counts from a certain value to 0
- Delay = (1sec/16 million(max value)) * #_of_cycles		(it counts 16mil each second)
- or: N * (1/SYSCLK) 
- Polling code: set a load time for each cycle (eg. 1 ms), have a while loop that stalls until cycle finishes, then forloop that for the desired time
Timers:
- Timer: internal in CPU. Counter: Extern thing connected to CPU
- When timer counts up to the value in the auto-reload_register, an interupt occurs and the timer starts back at 0 again
- Prescaler is basically gear reduction for timers. It divides the timer value before it reaches the counter, making it convenient
- update_event = timer/((prescaler+1)(period+1))

Day 6:
- Ouput compare: lets the timer hardware control the pin, without every needing the CPU. This means that you can generate a precise continuous signal (eg. PWM wavefore) while the CPU does other work
- In my code: Each hardware cycle, timer count is compared with target count in the compare register. When they match (default target is 0), hardware perform a pre-configured action to the output pin
- The action could be: set pin high, set pin low, do nothing, or toggle
- Input capture: pin's activity controls what gets written into the counter's associated register (without CPU). input capture lets the timer measure precise timing on a signal it doesn't control.
- You can choose to capture rising edge, falling edge, or both
Interrupts: 
- All things connect to the NVIC which handles interrupts, which then connects to the processor core
- Interrupts from the processor core are called exceptions
- GPIO pins are connected to EXTI lines (pin 0 of every port is connected to EXTI0_IRQ, or so forth for all other pins and ports. Thus, PA1 and PB1 cannot both be input interrupt pins because they share the same EXTI line)
- interrupt states: disabled (default), enabled, pending, active
- Priorities: Sets which interrupt should execute first, and which interrupt can interrupt which

Day 7:
- You set interrupts within a module: For example, if you want uart RX to send interrupts, you enable the interrupt peripheral in USART2.
- Then you enable that modules' interrupt in NVIC - basically just telling the NVIC to expect interrupts from that module. 
- The whole process is like you telling a mafia worker that they'll be sending messages over to the boss, and then you telling the boss to expect those messages from that worker
- However, if the interrupt, or exception in this case, is comming from inside the core, then you don't have to report it to NVIC (I'm pretty sure)
- Tip: When you change parameters of a function, remember to change them in the header file and function declarations!!

Day 8: (This week was extended)
- DMA allows data to be transfered in the background and only interrupts the processor when a whole data block is ready to by processed, resulting in no major impact on system performance
- DMA transfer: memory to peripheral, peripheral to memory, or memory to another memory
- DMA can automatically increment source and destination addresses after each transfer (eg. an array)
- DMA request mapping table shows every peripheral that has the DMA function and what channel it is

Day 9:
I2C:
- Transactions are initiated and completed by the master
- all msgs have an address frame and data frame
- data is placed on SDA line when SCL goes low and is sampled when SCL goes high 
- Open drain mode: pin can only actively pull to ground, instead of both high and low. Any device on the bus can pull the pin low. If no device is pulling it low, the external pullup resistors pull it back high.
- Rise time refers to how long it takes the electrical signal to transition from a low voltage level up to a high voltage level
- I2C read: First you have to write a slave address and a memory location TO THE SLAVE to tell it what to do. Once its recieved that and put its data where you specified, you then read it.
- the ADXL345 sends 16-bit values, but I2C only transfers 8 bits at a time. So each axis's full reading is split across two consecutive bytes. To combine them, shift the high byte left eight, keep the low byte as-is, the use bitwise OR to combine, as both bytes will have 0s on a half.
SPI:
- Basically the master and slave swap data.
- When master wants to send data, it pulls the SS line low. Then both lines start sending data at the same time! The first bit of the data differentiates the MOSI and MISO line (master-out-slave-in and master-in-slave-out).
- The data gets sent out and recieved in simultaneously like a queue on both ends, one bit at a time, the bits shifting until all data has been exchanged
- The clock is always constant
- CPOL: configure data capture/output occurs on the rising/falling edge. (sampling on the first vs second edge)	
- Always check if the module supports your protocol because I realised too late that the MPU6050 doesn't support SPI!
